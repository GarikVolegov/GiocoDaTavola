# Fase 0 — CI GitHub Actions — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Aggiungere una pipeline CI che esegue typecheck → lint → test → build su ogni push e PR, così i gate del progetto sono verificati automaticamente.

**Architecture:** Un singolo workflow GitHub Actions (`.github/workflows/ci.yml`) con un job che usa Node 20, installa con `npm ci` e lancia gli stessi script del `package.json` root. Nessun cambiamento al codice applicativo.

**Tech Stack:** GitHub Actions, Node 20, npm workspaces (server+client), vitest/eslint/tsc.

## Global Constraints

- Node 20 (il codice usa `loadEnvFile`, Node 20.12+).
- Gli script CI devono essere ESATTAMENTE quelli del `package.json` root: `npm run typecheck`, `npm run lint`, `npm test`, `npm run build`. Nessun nuovo script inventato.
- Nessun segreto nel workflow; nessuna dipendenza esterna (DB/Clerk/AI) richiesta — i test girano DB-less.
- Il workflow non deve introdurre comandi che falliscono in assenza di `DATABASE_URL`/Clerk (il progetto degrada a no-op senza).

### Task 1: Workflow CI

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: gli script root `typecheck`, `lint`, `test`, `build` (già esistenti in `package.json`).
- Produces: un workflow `CI` con job `build` che fallisce se un qualsiasi gate fallisce.

- [ ] **Step 1: Creare il workflow**

Scrivere `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: ['**']
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - name: Install
        run: npm ci
      - name: Typecheck
        run: npm run typecheck
      - name: Lint
        run: npm run lint
      - name: Test
        run: npm test
      - name: Build
        run: npm run build
```

- [ ] **Step 2: Validare che gli script esistano e siano verdi in locale**

Run: `npm run typecheck && npm run lint && npm test && npm run build`
Expected: tutti e quattro PASS (replica esatta dei job CI), come allo stato di partenza dell'audit (333 test verdi, build ok).

- [ ] **Step 3: Validare la sintassi YAML del workflow**

Run: `node -e "const fs=require('fs');const s=fs.readFileSync('.github/workflows/ci.yml','utf8');if(!/^name: CI/m.test(s)||!/npm ci/.test(s)||!/npm run build/.test(s)){throw new Error('workflow incompleto')};console.log('ci.yml ok')"`
Expected: stampa `ci.yml ok` (nessuna eccezione → il file contiene i passi chiave).

- [ ] **Step 4: Commit**

```bash

git commit -m "ci: workflow GitHub Actions (typecheck/lint/test/build su push+PR)"
```

- [ ] **Step 5: Push**

```bash

```
Expected: il branch è pushato; alla prossima push GitHub esegue il workflow.
