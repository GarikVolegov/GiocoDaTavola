## Codebase Patterns
- Stack: server/ (Node+Express+Socket.IO, TS) + client/ (React+Vite, TS), npm workspaces monorepo.
- Quality gate: `npm run typecheck && npm run lint && npm test && npm run build` from repo root.
- Server is authoritative; game state in-memory; votes are SECRET (only aggregate counts leave the server).
- Timers: server stores an expiry timestamp and broadcasts it; clients render the countdown locally.
- Server is CommonJS TS (module: CommonJS): use native __dirname, no .js import extensions needed. Client is ESM (Vite). Keep them separate.
- Quality gate is GREEN as of US-001: `npm run typecheck && npm run lint && npm test && npm run build`. Don't break it.
- ESLint flat config (eslint.config.js) uses typescript-eslint recommended (non type-checked) + no-unused-vars with ^_ ignore. Avoid `any` (no-explicit-any is an error).
- vitest config includes only `server/**/*.{test,spec}.ts` and has passWithNoTests. Put game-logic tests under server/src (e.g. server/src/game/__tests__).
- Client routing is path-based in App.tsx (`/host` -> HostApp, else PlayerApp); no react-router yet.
- Client socket: use `getSocket()` from client/src/shared/socket.ts â€” it's a memoized singleton, so StrictMode's double-mount and both views share ONE connection. Don't call io() directly.
- Shared client Socket.IO event names + payload types live in client/src/shared/events.ts (SocketEvents constants + payload interfaces), reused by host + player views. Server uses the same string literals in server/src/index.ts; keep them in sync.
- Server pattern for per-host state: map socket.id -> roomCode (hostRooms) so a re-emit returns the same room; clean up on 'disconnect'. Rooms themselves persist in the RoomStore for the process lifetime.
- Game logic modules go in server/src/game/*.ts with tests in server/src/game/__tests__/*.test.ts. Make randomness injectable (e.g. RoomStore(genCode)) so collisions/edge cases are testable deterministically.
- Adding a client dep: `npm install <pkg> --workspace client`. QR codes use `qrcode.react` (QRCodeSVG); wrap it in a white-background box for scannability in dark mode.
- `.last-branch` is a Ralph harness artifact; it's gitignored â€” don't commit it. `.vscode/` is also gitignored (editor config).
- Realtime broadcast: both host AND players `socket.join(code)`; broadcast aggregate state with `io.to(code).emit(event, payload)`. The lobby roster (nicknames) is PUBLIC â€” only VOTES are secret. A `broadcastLobby(code)` helper centralizes the emit.
- Player identity is the `socket.id` for now (US-015 reconnection will add a stable token). Player join dedups by socket.id like host rooms, so StrictMode's double-emit is harmless.
- Disconnect cleanup: keep a `playerRooms` map socket.id->code (mirrors `hostRooms`); on 'disconnect' remove the player and re-broadcast the lobby. Clean up BOTH maps.
- Client request/response over sockets: emit one event, server replies with a success event OR an error event (e.g. player:join -> player:joined | player:joinError). Map error CODES (not UI text) over the wire; keep a `JOIN_ERROR_MESSAGES` map in client/src/shared/events.ts for the Italian copy.
